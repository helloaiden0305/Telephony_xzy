# 通讯初始化最基础流程

> 从 Slot 配置到服务绑定 —— 一步一步看 Android 通讯服务是怎么启动的

---

## 📖 一、先看这个（新手友好）

### 一句话理解

> 手机能打电话、上网、发短信，是因为系统启动时做了一套**流水线作业**：
>
> **声明能力 → 识别 SIM 卡 → 服务就绪 → 分配编号 → 承接服务**

### 整体流程图

```
用户开机 / 插入 SIM 卡
        │
        ▼
┌─── 阶段 1 ────────────────────────────────────────────────┐
│  Slot 声明能力                                              │
│  "我这台手机有几个卡槽"                                      │
└─────────────────────────────────────────────────────────────┘
        │
        ▼
┌─── 阶段 2 ────────────────────────────────────────────────┐
│  SIM 卡插入 & 识别                                          │
│  "来了一张卡，我看看你是谁"                                   │
│  ──► Modem 通过 ATR 读取 ICCID / IMSI                       │
└─────────────────────────────────────────────────────────────┘
        │
        ▼
┌─── 阶段 3 ────────────────────────────────────────────────┐
│  服务应用就绪                                               │
│  "SIM 卡里的应用初始化完了，准备好干活了"                      │
│  ──► USIM 就绪 → 蜂窝通信可用                                │
│  ──► ISIM 就绪 → VoLTE / VoWiFi 可用                        │
└─────────────────────────────────────────────────────────────┘
        │
        ▼
┌─── 阶段 4 ────────────────────────────────────────────────┐
│  分配 subId                                                 │
│  "给你分配一个唯一编号，后面用这个编号来管理你"                 │
└─────────────────────────────────────────────────────────────┘
        │
        ▼
┌─── 阶段 5 ────────────────────────────────────────────────┐
│  服务绑定                                                   │
│  "Phone 接你的电话，Data 给你流量，SMS 帮你发短信"            │
└─────────────────────────────────────────────────────────────┘
        │
        ▼
   通讯服务就绪 ✅
```

### 5 秒记住这 5 步

| # | 阶段 | 大白话 | 类比 |
|---|------|--------|------|
| 1 | Slot 声明 | "我有几个卡槽" | 买房子：先有房间 |
| 2 | SIM 识别 | "你插了哪张卡" | 入住：登记身份 |
| 3 | 服务就绪 | "SIM 卡准备好了" | 通电通网：设施就绪 |
| 4 | subId 分配 | "给你发个房号" | 分配门牌号 |
| 5 | 服务绑定 | "服务接上你" | 物业入驻：水电网全通 |

## 阶段 1：Slot 声明与初始化（设备能力）

### 大白话

> 系统启动时，先问自己：**我这台设备有几个卡槽？** 这个数量是出厂就定好的，跟有没有插卡无关。

### 配置文件位置

```
典型配置路径：
• /vendor/etc/slot_config.xml
• /proc/device_tree/slot_info
• 或通过 properties 配置：persist.radio.multisim.config
```

### 示意图

```
┌─────────────────────────────────────────────────────────────────┐
│                      设备出厂 / 系统启动                          │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    Slot 配置文件                         │   │
│  │  ┌───────────┐  ┌───────────┐  ┌───────────┐          │   │
│  │  │  Slot 0   │  │  Slot 1   │  │  Slot 2   │  ...     │   │
│  │  │ (主卡槽)  │  │ (副卡槽)  │  │ (扩展槽)  │          │   │
│  │  └───────────┘  └───────────┘  └───────────┘          │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  说明：                                                          │
│  • 定义设备支持多少个逻辑通信通道                                │
│  • 与是否插入 SIM 卡无关                                        │
│  • 是系统管理多路通信能力的基石                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 核心代码

```java
// PhoneFactory.makeDefaultPhones()
int phoneCount = TelephonyManager.getDefault().getPhoneCount(); // Slot 数

for (int i = 0; i < phoneCount; i++) {
    Phone phone = new GsmPhone(
        context,
        CommandsInterface ril,
        notifier,
        i,                       // phoneId == slotIndex
        ...
    );
    sPhones[i] = phone;          // 固化 Phone-0 / Phone-1
}
```

### 完成标志

- ✅ Phone-0 / Phone-1 等实例创建完成
- ✅ 每个 Phone 绑定对应的 RIL 实例
- ✅ RIL 开始等待对应的 Radio HAL Service

### 失败影响

如果此阶段失败（如等待 HAL 超时），会直接导致：

- **Phone 进程启动 ANR**
- **SystemServer 启动阻塞**
- **整机通信能力夭折**

> ⚠️ 此阶段完全在 SystemServer 主线程执行，是启动流程中的高风险点。

## 阶段 2：SIM 卡插入与识别

### 大白话

> 用户把 SIM 卡插进卡槽 → Modem（基带芯片）通过 ATR 协议读取 SIM 卡信息 → 系统知道"来了一张什么卡"。

### ATR 读取流程

```
┌────────┐         ┌────────┐         ┌────────┐
│ SIM 卡  │         │ Modem  │         │ RIL    │
└───┬────┘         └───┬────┘         └───┬────┘
    │                  │                  │
    │  1. 上电复位     │                  │
    │◄─────────────────│                  │
    │                  │                  │
    │  2. ATR 响应     │                  │
    │─────────────────►│                  │
    │     (Answer To Reset)              │
    │                  │                  │
    │  3. 读取 ICCID   │                  │
    │◄─────────────────│                  │
    │     读取 IMSI    │                  │
    │◄─────────────────│                  │
    │     读取 GID     │                  │
    │◄─────────────────│                  │
    │                  │                  │
    │                  │  4. 上报 RIL     │
    │                  │─────────────────►│
    │                  │     RIL_UNSOL_RESPONSE_SIM_STATUS_CHANGED
```

### ATR 读取的信息

| 信息类型 | 说明 | 用途 |
|---------|------|------|
| **ICCID** | SIM 卡唯一标识 | 识别 SIM 卡身份 |
| **IMSI** | 国际移动用户识别码 | 网络注册用 |
| **GID** | 组标识符 | 运营商分组 |
| **电压等级** | SIM 卡支持的电压 | 5V / 3V / 1.8V 自适应 |

### UICC 对象结构

RIL 根据 Modem 读取到的信息，构建一个 **UICC（Universal Integrated Circuit Card）** 对象，代表 SIM 卡的逻辑视图：

```
┌─────────────────────────────────────────────────────────────────┐
│                      UICC 对象（逻辑视图）                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    文件系统                              │   │
│  │  ┌───────────┐  ┌───────────┐  ┌───────────┐          │   │
│  │  │   MF      │  │   DF      │  │   EF      │          │   │
│  │  │ (主文件)   │  │ (专用文件) │  │ (基本文件) │          │   │
│  │  └───────────┘  └───────────┘  └───────────┘          │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    应用列表                              │   │
│  │  ┌───────────┐  ┌───────────┐  ┌───────────┐          │   │
│  │  │   USIM    │  │   ISIM    │  │   CSIM    │  ...     │   │
│  │  │ (蜂窝通信) │  │ (IMS 服务) │  │ (CDMA)    │          │   │
│  │  └───────────┘  └───────────┘  └───────────┘          │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 关键组件

| 组件 | 职责 |
|------|------|
| `UiccController` | UICC 的总控制器，管理所有 Slot 的 UICC 实例 |
| `UiccCard` | 代表一张物理 SIM 卡 |
| `UiccSlot` | 代表一个逻辑卡槽 |
| `UiccPort` | 代表卡槽上的物理触点（多端口场景） |

### 状态流转

```
ABSENT → LOADING → LOADED → READY
  ↑          ↑         ↑        ↑
 无卡     读取中    已加载   可服务
```

## 阶段 3：服务应用就绪（USIM / ISIM 等）

### 大白话

> SIM 卡插好了，但卡上本身装着不同的"应用"（USIM、ISIM 等），这些应用要先初始化，才能真正提供服务。

### 主要服务应用

| 应用类型 | 用途 | 标识符 |
|---------|------|--------|
| **USIM** | 蜂窝通信（2G/3G/4G/5G） | EF_USIM |
| **ISIM** | IMS 服务（VoLTE、VoWiFi） | EF_ISIM |
| **CSIM** | CDMA 网络（部分运营商） | EF_CSIM |
| **SIM** | 2G 传统网络 | EF_SIM |

### 服务就绪流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    服务应用初始化流程                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. UICC 构建完成                                               │
│         │                                                       │
│         ▼                                                       │
│  2. 应用选择（SELECT ADF）                                      │
│         │                                                       │
│         ▼                                                       │
│  3. PIN 码验证（可选）                                          │
│         │                                                       │
│         ▼                                                       │
│  4. 认证密钥协商                                                 │
│         │                                                       │
│         ▼                                                       │
│  5. 服务注册（IMS/CS）                                          │
│         │                                                       │
│         ▼                                                       │
│  6. 服务就绪状态                                                 │
│         │                                                       │
│         ▼                                                       │
│  7. 通知 Framework（RIL_UNSOL_SERVICE_STATE_CHANGED）          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 初始化内容

1. **PIN 码验证**：如用户启用了 PIN 码锁，需先验证通过
2. **应用选择（SELECT）**：通过 APDU 命令选择对应的应用（AID）
   - 如 ISIM 的 AID 为 `A0000000871002FFFFFFFF8900000100`
3. **文件读取**：读取 EF_IMPI、EF_IMPU（IMS 标识）、EF_DOMAIN（IMS 域名）等
4. **状态上报**：向 Framework 上报应用就绪状态

### 完成标志

- ✅ USIM 应用就绪 → 蜂窝通信可用
- ✅ ISIM 应用就绪 → IMS / VoLTE 可用
- ✅ 所有必要文件读取完成

### 与 IMS 的关联

```
ISIM 就绪
  ↓
ImsService 检测到 ISIM 状态变化
  ↓
ImsManager 触发 IMS 注册流程
  ↓
向 IMS 核心网发起 REGISTER
```

> ⚠️ 如果 ISIM 未就绪，IMS 注册无法进行，VoLTE / VoWiFi 不可用。

### 服务状态

| 状态 | 说明 | 后续动作 |
|------|------|---------|
| **LOST** | SIM 卡丢失 | 等待重新插入 |
| **LOADING** | 加载中 | 等待初始化完成 |
| **LOADED** | 已加载 | 继续服务应用初始化 |
| **READY** | 就绪 | 分配 subId，绑定服务 |
| **PIN_REQUIRED** | 需要 PIN 码 | 等待用户输入 |
| **PERM_DISABLED** | 永久禁用 | 需 PUK 码解锁 |

## 阶段 4：SubscriptionManager 分配 subId

### 大白话

> SIM 卡准备好了，系统给它分配一个**唯一编号**（subId），后面所有的服务都通过编号来找这张卡，而不是通过物理卡槽。

### 关键概念辨析

| 标识符 | 含义 | 是否可变 | 示例 |
|--------|------|---------|------|
| `slotIndex` | 物理卡槽编号 | 固定 | 0, 1 |
| `phoneId` | Phone 实例编号 | 固定 | 0, 1 |
| `subId` | 订阅实例编号 | **可变** | 1, 2, 3... |

> **slotIndex 是"位置"，subId 是"服务"。**
>
> - 一张 SIM 卡从 Slot 0 拔到 Slot 1，`slotIndex` 变了，但它还是同一个订阅（`subId` 可能不变）
> - 同一张 SIM 卡在不同时间插入，`subId` 可能不同（取决于系统策略）

### subId 分配流程

```
Slot 0 + SIM 卡 A  ──►  UICC 就绪  ──►  subId = 1

Slot 1 + SIM 卡 B  ──►  UICC 就绪  ──►  subId = 2

说明：
• subId 是动态分配的，与 slotIndex 无固定绑定关系
• SIM 卡拔出后，subId 可能被回收
• 上层服务通过 subId 访问具体 SIM 卡的服务能力
```

```
UiccController 检测到 UICC 就绪
  ↓
发送 ACTION_SIM_STATE_CHANGED 广播
  ↓
SubscriptionController 监听广播
  ↓
检查是否需要创建/更新 SubscriptionInfo
  ↓
分配唯一的 subId
  ↓
写入 TelephonyProvider 数据库
```

### 核心代码

```java
// SubscriptionController.java
public void addSubscriptionInfo(String iccId, int slotIndex, int[] subId) {
    // 生成唯一的 subId
    subId[0] = getNextSubId();

    // 创建 SubscriptionInfo 对象
    SubscriptionInfo subInfo = new SubscriptionInfo(
        subId[0],
        iccId,
        slotIndex,
        ...
    );

    // 写入数据库
    mTelephonyProvider.insertSubscription(subInfo);
}
```

### 数据库存储

TelephonyProvider 中的 `siminfo` 表存储所有订阅信息：

```sql
CREATE TABLE siminfo (
    _id INTEGER PRIMARY KEY,      -- subId
    icc_id TEXT,                   -- ICCID
    slot_index INTEGER,            -- Slot 编号
    display_name TEXT,             -- 用户可见名称
    ...
);
```

## 阶段 5：服务绑定（Phone / Data / SMS）

### 大白话

> subId 分配好了，最后一步：让各个通讯服务"接上"这个编号，正式上岗。

```
                      subId（唯一订阅标识）
                        │
        ┌───────────────┼───────────────┐
        │               │               │
        ↓               ↓               ↓
    Phone           Data            SMS
  （通话服务）    （数据服务）       （短信服务）
```

### 各服务职责

#### 1. Phone（通话服务）

- **类**：`com.android.internal.telephony.Phone`
- **职责**：管理语音通话、呼叫状态、服务状态
- **绑定方式**：通过 `phoneId` 关联到对应的 `subId`

```java
// GsmPhone.java
public class GsmPhone extends Phone {
    private int mPhoneId;      // 对应 slotIndex
    private int mSubId;        // 绑定 subId

    public void onSubscriptionInfoChanged() {
        // 处理订阅信息变化
    }
}
```

#### 2. Data（数据服务）

- **类**：`DataNetworkController`
- **职责**：管理移动数据连接、APN 配置、数据开关
- **绑定方式**：通过 `OnSubscriptionsChangedListener` 监听

```java
// DataNetworkController.java
mSubscriptionManager.addOnSubscriptionsChangedListener(
    new OnSubscriptionsChangedListener() {
        @Override
        public void onSubscriptionsChanged() {
            // 重新评估数据网络
            updateDataNetwork();
        }
    }
);
```

#### 3. SMS（短信服务）

- **类**：`SmsManager`
- **职责**：管理短信发送/接收、存储空间、默认 SIM 选择
- **绑定方式**：通过 `getSmsManagerForSubscriptionId(subId)`

```java
// SmsManager.java
public static SmsManager getSmsManagerForSubscriptionId(int subId) {
    return new SmsManager(subId);
}
```

### 代码示例

```java
// 获取 Phone 服务实例（绑定到特定 subId）
Phone phone = PhoneFactory.getPhone(subId);

// 获取数据服务状态
boolean enabled = DataResolver.isEnabled(subId);

// 发送短信
SmsManager smsManager = SmsManager.getSmsManagerForSubscriptionId(subId);
smsManager.sendTextMessage(...);

// 注册服务状态监听
TelephonyManager tm = TelephonyManager.from(context);
tm.listen(phoneStateListener, PhoneStateListener.LISTEN_SERVICE_STATE);
```

### 服务绑定完成标志

- ✅ Phone 服务可发起/接听通话
- ✅ Data 服务可建立移动数据连接
- ✅ SMS 服务可发送/接收短信

## 📖 四、关键依赖关系

### 严格的前置条件

| 阶段 | 前置条件 | 失败后果 |
|------|---------|---------|
| 阶段 1 | 设备声明 Slot 数 | 无法创建 Phone，ANR |
| 阶段 2 | Slot 初始化完成 | 无法识别 SIM 卡 |
| 阶段 3 | UICC 加载完成 | 服务应用无法初始化 |
| 阶段 4 | 服务应用就绪 | 无法分配 subId |
| 阶段 5 | subId 分配成功 | 通讯服务无法绑定 |

### 异步 vs 同步

| 阶段 | 执行方式 | 阻塞风险 |
|------|---------|---------|
| 阶段 1 | **同步（主线程）** | 🔴 高（ANR 风险） |
| 阶段 2 | 异步（后台线程） | 🟡 中 |
| 阶段 3 | 异步（后台线程） | 🟡 中 |
| 阶段 4 | 异步（广播处理） | 🟢 低 |
| 阶段 5 | 异步（广播监听） | 🟢 低 |

## 📖 五、常见问题排查

### 问题 1：SIM 卡显示"无服务"

**可能原因：**

- 阶段 1 失败：Slot 未初始化（检查 Radio HAL）
- 阶段 2 失败：UICC 未加载（检查 SIM 卡接触）
- 阶段 3 失败：USIM 应用未就绪（检查 PIN 码）

**排查步骤：**

```bash
# 检查 Slot 状态
adb shell dumpsys iphonesubinfo

# 检查 UICC 状态
adb shell dumpsys uicc

# 检查订阅信息
adb shell dumpsys subscription
```

### 问题 2：VoLTE 不可用

**可能原因：**

- 阶段 3 失败：ISIM 应用未就绪
- 阶段 4 失败：subId 未正确分配
- 阶段 5 失败：ImsService 未绑定 subId

**排查步骤：**

```bash
# 检查 IMS 状态
adb shell dumpsys ims

# 检查 ImsService
adb shell dumpsys activity service ImsService
```

### 问题 3：双卡只有一张卡工作

**可能原因：**

- Slot1 的 Radio HAL 未启动
- 阶段 1 只完成了 Slot0 的初始化
- 阶段 4 只分配了一个 subId

**排查步骤：**
